⚠️ Escape, Not Ctrl+C

Always use Escape to stop Claude, not Ctrl+C. Ctrl+C terminates the entire process, while Escape gracefully interrupts the current operation.

Quick Workflows

The Hash Key Trick

Press # to quickly add notes to your CLAUDE.md without leaving the conversation:

# Type # then your note
# "Remember: use camelCase for variables in this project"

# Claude adds it to CLAUDE.md automatically

Plan Mode Pattern

# 1. Enter plan mode
> (press Shift+Tab)

# 2. Describe what you want
> "I need to refactor the auth module to use JWT"

# 3. Claude creates a plan without executing

# 4. Review and approve/modify

# 5. Exit plan mode to execute
> (press Shift+Tab again)

Creating Custom Skills

Quick Skill Setup

# Create a skill directory
mkdir -p .claude/skills/my-skill

# Create SKILL.md with frontmatter
cat > .claude/skills/my-skill/SKILL.md << 'EOF'
---
name: my-skill
description: Brief description for auto-invocation
allowed-tools: Grep, Read, Glob
user-invocable: true
model: haiku
---

# My Custom Skill

Instructions for what the skill should do...

## Usage
/my-skill [arguments]
EOF

# Now use it!
/my-skill

Skill Frontmatter Cheatsheet

---
name: skill-name          # Required: invoked as /skill-name
description: ...          # Required: helps auto-invocation
allowed-tools: A, B, C    # Optional: restrict tools
user-invocable: true      # Optional: allow /skill-name
model: haiku|sonnet|opus  # Optional: override model
---

Real-World Skill Example

# .claude/skills/cw-search/SKILL.md
---
name: cw-search
description: Search Cover Whale knowledge base and codebase
allowed-tools: Grep, Read, Glob
user-invocable: true
model: haiku
---

# Cover Whale Search Skill

## Search Strategy
1. Use Grep with files_with_matches first
2. Prioritize documentation/ and knowledge-base/
3. Read only top 2-3 relevant files
4. Return file:line references

## Response Format
- Direct answer first
- Source file references
- Offer to read more if needed

# Usage examples:
# /cw-search billing process
# /cw-search kubernetes setup
💡 Skill Tips
  • Use model: haiku for search/lookup (fast & cheap)
  • Use model: sonnet for code generation
  • Restrict tools to minimize risk and tokens
  • Put team skills in .claude/skills/ (git-tracked)

The @ Symbol

The @ symbol is Gemini's file reference syntax. It's faster than explicitly asking to read files.

# Reference single file
> @src/app.ts "What does this file do?"

# Reference multiple files
> @src/auth/*.ts "How does authentication work?"

# Reference with glob patterns
> @packages/**/index.ts "List all package entry points"

# Reference in context
> @package.json @tsconfig.json "Are these configurations aligned?"
💡 @ Tip

Use Tab after @ to autocomplete file paths. Gemini indexes your project for fast suggestions.

Checkpointing

Gemini's checkpoint system lets you save and resume sessions, which is invaluable for long-running tasks.

# Save checkpoint before risky operation
> /save before-refactor

# Do some work...

# If things go wrong, restore
> /load before-refactor

# Save final state
> /save after-refactor-complete

Token Caching

Gemini 3 features improved token caching for repeated contexts, making subsequent queries faster and cheaper. With the 2M token context of Gemini 3 Pro, caching becomes even more valuable.

# First query loads files (slower, uses more tokens)
> @src/**/*.ts "Explain the architecture"

# Subsequent queries reuse cached context (faster)
> "Now, what about error handling?"
> "And how is logging implemented?"

# Check token usage
> /tokens
💰 Cost Optimization

Load large contexts once at the start of a session, then ask multiple questions. Cached tokens are significantly cheaper.

Headless Mode

Run Gemini without interactive mode for automation:

# Single query
gemini "Explain the main function in @src/index.ts"

# With file output
gemini "Generate tests for @src/utils.ts" > tests.ts

# In scripts
#!/bin/bash
gemini "Review @$1 for security issues" | tee review.md

CI/CD Integration

# In GitHub Actions
- name: AI Code Review
  run: |
    gemini "Review the changes in this PR for issues" \
      --diff "${{ github.event.pull_request.diff_url }}"
