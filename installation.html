📦 Installation Guide

Get both Claude Code and Gemini CLI set up on your machine. Choose your platform below for tailored installation instructions.

Prerequisites

  • Claude Pro/Max account or Console access - For authentication
  • macOS 13.0+, Ubuntu 20.04+/Debian 10+, or Windows 10 1809+
  • 4 GB+ RAM and internet connection
💡 Native Installation Recommended

Claude Code has native installers for all platforms that auto-update in the background. No Node.js required!

Installation

Option 1: Native Installer (Recommended)

Open Terminal and run:

curl -fsSL https://claude.ai/install.sh | bash
✅ Auto-Updates Included

The native installer automatically updates Claude Code in the background. No manual updates needed!

Option 2: Homebrew

brew install --cask claude-code
⚠️ Homebrew Note

Homebrew installations do not auto-update. Run brew upgrade claude-code periodically to get the latest version.

Option 3: npm (Deprecated)

# Only if native installer doesn't work
npm install -g @anthropic-ai/claude-code
💡 Windows Native Support

Claude Code now has native Windows support - no WSL required! Requires Git for Windows for Git Bash.

Option 1: PowerShell (Easiest)

  1. Open PowerShell as Administrator (Press Win + X, select "Windows PowerShell (Admin)")
  2. Run the installation command:
irm https://claude.ai/install.ps1 | iex
⚠️ Execution Policy Error?

If you get an execution policy error, first run:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Then retry the installation command.

Option 2: Command Prompt

  1. Open Command Prompt (Press Win + R, type cmd, press Enter)
  2. Run this command:
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

Option 3: WinGet

winget install Anthropic.ClaudeCode

Note: WinGet installations do not auto-update. Run winget upgrade Anthropic.ClaudeCode to update.

Option 4: npm (Deprecated)

# Only if native installer doesn't work (requires Node.js 18+)
npm install -g @anthropic-ai/claude-code

Platform-Specific Troubleshooting

"claude command not found"

Add Claude Code to your PATH:

# Add to ~/.zshrc or ~/.bashrc
export PATH="$HOME/.local/bin:$PATH"

# Reload your shell
source ~/.zshrc  # or source ~/.bashrc

Permission Denied on Install

# Fix permissions for local bin directory
mkdir -p ~/.local/bin
chmod 755 ~/.local/bin

Authentication Issues

# Reset credentials
rm -rf ~/.claude
rm ~/.claude.json

# Restart Claude Code
claude

Verify Installation

# Check installation type and version
claude doctor

"claude command not found"

The installer should add Claude Code to your PATH automatically. If not:

  1. Press Win + Pause/Break (or Settings > System > About > Advanced system settings)
  2. Click "Environment Variables"
  3. Under "User variables", find or create PATH
  4. Add: %USERPROFILE%\.local\bin
  5. Click OK and restart your terminal

"Git Bash not found"

  1. Install Git for Windows
  2. Find your Git Bash installation (usually C:\Program Files\Git\bin\bash.exe)
  3. Set the environment variable in PowerShell:
[System.Environment]::SetEnvironmentVariable('CLAUDE_CODE_GIT_BASH_PATH', 'C:\Program Files\Git\bin\bash.exe', 'User')
  1. Restart your terminal

Remove-Item -Path "$env:USERPROFILE\.claude.json" -Force

🛠 General Troubleshooting

Common Issues (All Platforms)

Node.js Version Too Old (for npm installs)

# Check your version
node --version

# Update using Homebrew:
brew install node@20

# Or using nvm:
nvm install 20
nvm use 20

Permission Denied on npm Install

# DO NOT use sudo with npm! Fix permissions instead:
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'

# Add to ~/.zshrc:
export PATH=~/.npm-global/bin:$PATH
source ~/.zshrc

Homebrew Issues

# Update Homebrew
brew update

# Fix permissions
sudo chown -R $(whoami) /usr/local/Homebrew

# Reinstall if needed
brew reinstall claude-code

Node.js Version Too Old (for npm installs)

# Check your version
node --version

# Download latest from nodejs.org
# Or use nvm-windows:
nvm install 20
nvm use 20

npm Global Install Location

# Find npm global bin path
npm config get prefix

# Default is: %APPDATA%\npm
# Add this to your PATH if commands aren't found

PowerShell Execution Policy

# If scripts won't run:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Node.js Version Too Old (for npm installs)

# Check your version
node --version

# Using nvm (recommended):
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20

# Or using NodeSource:
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

Permission Denied on npm Install

# DO NOT use sudo with npm! Fix permissions instead:
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'

# Add to ~/.bashrc:
export PATH=~/.npm-global/bin:$PATH
source ~/.bashrc

Snap/Flatpak Node.js Issues

Snap and Flatpak versions of Node.js may have permission issues. Use nvm or system packages instead.

Authentication Fails (All Platforms)

  • Check your API key is correct
  • Ensure environment variable is set in current shell
  • For Gemini: Try gemini auth logout && gemini auth login
  • For Claude: Delete auth files and restart (see platform-specific troubleshooting above)
