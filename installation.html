Option A: Claude Desktop App (Easiest - No Terminal Required)

🌟 Recommended for Non-Technical Users

If you're not comfortable with terminal/command line installations, the Claude Desktop App is the easiest way to use Claude Code. It's a regular application you download and install like any other program - no terminal commands needed!

What is the Claude Desktop App?

The Claude Desktop App is a standalone application that gives you access to Claude Code through a friendly graphical interface. You can chat with Claude, have it read and edit files, and perform all the same tasks as the terminal version - but with a visual window instead of typing commands.

Benefits of the Desktop App

  • No terminal required - Works like a normal desktop application
  • Simple installation - Download, install, and sign in
  • Visual interface - See your conversations in a clean, easy-to-read format
  • Same powerful features - Can read files, write code, and help with all the same tasks
  • Auto-updates - Stays up to date automatically

Installing Claude Desktop on Mac

  1. Download the app: Go to https://claude.ai/download
  2. Open the installer: Double-click the downloaded .dmg file
  3. Drag to Applications: Drag the Claude icon to your Applications folder
  4. Open Claude: Find Claude in your Applications folder (or use Spotlight: Cmd + Space, type "Claude")
  5. Sign in: Log in with your Claude account (the same one IT approved for you)
  6. Start using Claude: You're ready! Type your questions or requests in the chat window
🍎 Mac Security Note

If you see a message saying the app "can't be opened because it's from an unidentified developer," go to System Preferences > Security & Privacy and click "Open Anyway."

Installing Claude Desktop on Windows

  1. Download the app: Go to https://claude.ai/download
  2. Run the installer: Double-click the downloaded .exe file
  3. Follow the prompts: Click "Next" through the installation wizard
  4. Launch Claude: Find Claude in your Start Menu or on your Desktop
  5. Sign in: Log in with your Claude account (the same one IT approved for you)
  6. Start using Claude: You're ready! Type your questions or requests in the chat window
💻 Windows Security Note

If Windows SmartScreen shows a warning, click "More info" and then "Run anyway". This is normal for newly downloaded applications.

Enabling Claude Code Features in the Desktop App

🛠 IMPORTANT: Setting Up Your PATH (If "claude" Command Not Found)

After installation, if you get a "claude is not recognized" or "command not found" error, you need to add Claude Code to your system PATH. This tells Windows where to find the Claude program.

Step-by-Step PATH Setup:

  1. Open System Properties:
    • Press Win + R to open the Run dialog
    • Type sysdm.cpl and press Enter
    • Click the "Advanced" tab
    • Click the "Environment Variables..." button at the bottom
  2. Edit the PATH variable:
    • In the "User variables" section (top half), find and select "Path"
    • Click "Edit..."
    • Click "New"
    • Add this path: %USERPROFILE%\.local\bin
    • Click "OK" to close each window
  3. Restart your terminal:
    • Close ALL open PowerShell, Command Prompt, or terminal windows
    • Open a new PowerShell or Command Prompt window
  4. Verify it works: 
    claude --version

    You should see the Claude Code version number (e.g., claude 1.x.x)

Alternative: Quick PATH Fix via PowerShell

If you prefer using commands, run this in PowerShell as Administrator:

# Add Claude Code to your PATH permanently
$currentPath = [Environment]::GetEnvironmentVariable("Path", "User")
$newPath = "$env:USERPROFILE\.local\bin"
if ($currentPath -notlike "*$newPath*") {
    [Environment]::SetEnvironmentVariable("Path", "$currentPath;$newPath", "User")
    Write-Host "PATH updated! Please restart your terminal."
} else {
    Write-Host "PATH already contains Claude Code location."
}

Then close and reopen your terminal window.

"claude command not found" or "not recognized"

This is the most common issue on Windows. Follow these steps:

Step 1: Verify the Installation Location

First, check if Claude Code was installed:

# Open PowerShell and run:
Test-Path "$env:USERPROFILE\.local\bin\claude.exe"

# If this returns "True", Claude is installed but not in your PATH
# If "False", the installation may have failed - try reinstalling

Step 2: Add Claude to Your PATH

  1. Press Win + R, type sysdm.cpl, press Enter
  2. Click "Advanced" tab, then "Environment Variables..."
  3. Under "User variables", find or create Path
  4. Click "Edit...", then "New"
  5. Add: %USERPROFILE%\.local\bin
  6. Click OK on all windows
  7. Close ALL terminal windows and open a new one

Step 3: Verify It Works

claude --version

"Git Bash not found"

Claude Code requires Git Bash. If you see this error:

  1. Install Git for Windows (see Prerequisites section above)
  2. Find your Git Bash installation (usually C:\Program Files\Git\bin\bash.exe)
  3. Set the environment variable in PowerShell:
[System.Environment]::SetEnvironmentVariable('CLAUDE_CODE_GIT_BASH_PATH', 'C:\Program Files\Git\bin\bash.exe', 'User')
  1. Restart your terminal

# PowerShell

Remove-Item -Path "$env:USERPROFILE\.claude.json" -Force
