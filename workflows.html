Explore - Plan - Execute - Commit

The fundamental Claude workflow for any development task.

1. Explore

Understand the codebase and current state

/add relevant/files.ts
"Explain how X works"

2. Plan

Design the solution before implementing

# Enter plan mode
(Shift+Tab)
"Create a plan to..."

3. Execute

Implement the changes incrementally

# Exit plan mode
(Shift+Tab)
"Execute step 1"

4. Commit

Review and commit changes

"Review changes and
create a commit message"

Checklist-Driven Development

Break complex tasks into explicit checklists for reliable execution.

# Start with a checklist prompt
"I need to add user authentication. Create a checklist:

[ ] Create User model
[ ] Add password hashing utility
[ ] Create auth middleware
[ ] Add login/logout routes
[ ] Add session handling
[ ] Create tests

Execute each item one at a time. After completing each,
show me the changes and mark it done. Wait for my approval
before moving to the next item."
✅ Checklist Benefits
  • Clear progress tracking
  • Easy to pause and resume
  • Natural rollback points
  • Prevents scope creep

Debugging Workflow

# Step 1: Share the problem
"I'm seeing this error: [paste error]
It happens when: [describe conditions]"

# Step 2: Add relevant context
/add suspected/file.ts
/add related/file.ts

# Step 3: Guided investigation
"Walk me through possible causes.
Don't fix anything yet, just analyze."

# Step 4: Confirm and fix
"Your theory about X seems right.
Please implement the fix."

# Step 5: Verify
"Now run the tests to verify the fix"

Test-Driven Development

# Step 1: Write tests first
"I need a function that calculates compound interest.
Write the tests first:
- Basic calculation
- Edge cases (zero principal, zero rate)
- Large numbers
- Negative values (should throw)

Use Vitest syntax."

# Step 2: Run tests (they should fail)
"Run the tests to confirm they fail"

# Step 3: Implement
"Now implement the function to make all tests pass"

# Step 4: Verify
"Run tests again to confirm they pass"

Session-Based Development

Use Gemini's checkpointing for long-running tasks.

# Start new feature session
> /save feature-user-profile-start

# Work on the feature
> @src/models/user.ts "Add profile fields"
> "Create profile API endpoints"
> "Add validation"

# Save progress checkpoint
> /save feature-user-profile-v1

# Next day, resume
> /load feature-user-profile-v1
> "Continue where we left off"

# Save final version
> /save feature-user-profile-complete
💾 Session Tips
  • Save before risky operations
  • Use descriptive session names
  • Save at natural breakpoints
  • Clean up old sessions periodically

Migration Workflow

# Step 1: Analyze current state
> @**/*.ts
"Identify all uses of the old API pattern:
fetch('/api/...')"

# Step 2: Create migration script
"Generate a migration that:
1. Finds all matching patterns
2. Replaces with new httpClient calls
3. Preserves error handling
4. Adds type annotations"

# Step 3: Execute with verification
"Execute the migration. After each file:
1. Run TypeScript compiler
2. Run relevant tests
Stop if any failures occur."

# Step 4: Generate report
"Generate migration report:
- Files changed
- Patterns replaced
- Any manual fixes needed"

The Relay Race

The ultimate workflow: combine strengths of both tools.

🏆 The Handoff Protocol
  1. Claude analyzes and creates a plan
  2. Gemini executes the plan broadly
  3. Claude reviews the results
  4. Repeat until complete

Phase 1: Analysis (Claude)

# Claude analyzes the problem
/add src/legacy/payment.ts

"This legacy payment module needs modernization.
Analyze and create a detailed plan in PLAN.md covering:
1. Current architecture issues
2. Proposed new structure
3. Migration steps
4. Risk assessment
5. Rollback procedures"

Phase 2: Execution (Gemini)

# Gemini executes the plan
> @PLAN.md @src/legacy/payment.ts

"Execute the plan steps 1-3:
1. Create new module structure
2. Migrate core functions
3. Add new tests

Update PLAN.md with progress after each step."

Phase 3: Review (Claude)

# Claude reviews the changes
/add PLAN.md
"Show me git diff since last commit"

"Review all changes against the original plan:
1. Are the changes correct?
2. Any missed edge cases?
3. Test coverage adequate?
4. Ready for step 4?"

The Supervisor Pattern

Use a lightweight agent to maintain high-level progress tracking.

# Supervisor (separate chat or window)
# Maintains MASTER_TASKS.md with overall progress

# Worker agents report to supervisor
# After completing a task:

# In Claude session:
"Task complete. Update HANDOFF.md with:
- What was done
- Current state
- Next recommended step"

# Supervisor reviews HANDOFF.md and assigns next task
# to appropriate worker (Claude or Gemini)
